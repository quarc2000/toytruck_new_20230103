# QMC5883L (GY-271)

## Summary
- 3-axis digital magnetometer with I2C interface.
- Default 7-bit I2C address is `0x0D`.
- Intended use here is heading support and magnetic-disturbance detection.

## Datasheet
- Local file: `.context/resources/datasheets/13-52-04 QMC5883L Datasheet Rev. B.pdf`
- Original source: `https://www.qstcorp.com/upload/pdf/202512/13-52-04%20QMC5883L%20Datasheet%20Rev.%20B.pdf`

## Facts We Rely On
- Communication is I2C, standard or fast mode, up to `400 kHz`.
- The device powers up into standby mode after POR.
- Default 7-bit I2C address is `0x0D`.
- Chip ID register is `0x0D` and returns `0xFF`.
- Magnetic output registers are:
  - `0x00` X LSB
  - `0x01` X MSB
  - `0x02` Y LSB
  - `0x03` Y MSB
  - `0x04` Z LSB
  - `0x05` Z MSB
- Axis output is 16-bit two's-complement, little-endian:
  - `value = (lsb) | (msb << 8)`
- Status register is `0x06`:
  - bit `0` = `DRDY`
  - bit `1` = `OVL`
  - bit `2` = `DOR`
- Control register 1 is `0x09`:
  - bits `1:0` = mode
  - bits `3:2` = ODR
  - bits `5:4` = range
  - bits `7:6` = OSR
- Control register 2 is `0x0A`:
  - bit `7` = soft reset
  - bit `6` = pointer roll-over enable
  - bit `0` = interrupt enable control
- Set/reset period register is `0x0B`.
- Datasheet recommendation: write `0x0B = 0x01`.
- Datasheet continuous-mode example:
  - write `0x0B = 0x01`
  - write `0x09 = 0x1D`
- `0x1D` means:
  - OSR = `512`
  - full scale = `8 G`
  - ODR = `200 Hz`
  - mode = continuous
- For the current truck debug service we intentionally use `0x11` instead:
  - same continuous mode
  - same `8 G`
  - same `OSR 512`
  - lower `10 Hz` ODR so a human-readable polling task does not sit in constant data-overrun

## Read Sequence
- Normal read flow from the datasheet:
  1. Check `DRDY` pin or poll `0x06` bit `0`
  2. Read `0x00` through `0x05`
  3. Reading any data register clears `DRDY` and `DOR`
  4. Data stays locked until `0x05` is read
- Practical consequence:
  - read the full `0x00..0x05` block in one transaction
  - do not read only part of the XYZ block

## Setup Sequence We Should Trust
- After power-on, allow POR completion.
- Optional soft reset: write `0x0A = 0x80`.
- Write `0x0B = 0x01`.
- Write `0x09 = 0x1D` for the datasheet's continuous-mode example.
- Verify:
  - chip ID at `0x0D`
  - control register `0x09`
  - set/reset register `0x0B`

## Important Clarifications
- `0x06` is a status register, not a setup register.
- Writing to `0x06` is not part of the datasheet setup flow.
- The device already applies internal offset cancellation and temperature compensation in continuous mode.
- Temperature registers exist at `0x07` and `0x08`, but temperature is not needed for first heading bring-up.

## Project Usage
- Current project task uses the QMC5883L through the TCA9548 expander path on mux port `3`.
- Current truck-heading assumption for the active service path is:
  - sensor `+Y` = forward along the truck body
  - sensor `-X` = truck-right lateral
  - `calculatedMagCourse` is therefore currently derived from `atan2(-X, Y)`
- Current shared-variable targets are:
  - `rawMagX`
  - `rawMagY`
  - `rawMagZ`
  - `calculatedMagCourse`
  - `calculatedMagDisturbance`

## Known Quirks
- If byte order is wrong, signed axis values will look erratic or nearly constant.
- If the full `0x00..0x05` block is not read, data lock behavior can make updates look stale.
- Nearby ferrous parts, copper, wiring, batteries, and motors can distort heading badly even when the sensor itself is working.
- `OVL` indicates overflow, and `DOR` indicates data was skipped.
