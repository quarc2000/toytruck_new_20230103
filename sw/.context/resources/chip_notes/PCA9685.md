# PCA9685

## Summary

The PCA9685 is a 16-channel PWM controller with an internal oscillator, per-channel 12-bit timing registers, and a shared prescaler that sets the PWM frame frequency.

For chip-level bring-up, the important topics are `MODE1`, `MODE2`, `PRESCALE`, the per-channel LED register layout, sleep and restart sequencing, and the need for auto-increment during multi-byte register access.

## Facts We Rely On

- Default I2C base address is `0x40`
- `MODE1` is register `0x00`
- `MODE2` is register `0x01`
- `LED0_ON_L` starts at register `0x06`
- `PRESCALE` is register `0xFE`
- `SLEEP` in `MODE1` is bit `0x10`
- `AI` auto-increment in `MODE1` is bit `0x20`
- `RESTART` in `MODE1` is bit `0x80`
- `OUTDRV` push-pull output mode in `MODE2` is bit `0x04`
- Each output channel uses 4 registers:
  - `LEDn_ON_L`
  - `LEDn_ON_H`
  - `LEDn_OFF_L`
  - `LEDn_OFF_H`
- `FULL_OFF` is bit `0x10` in `LEDn_OFF_H`

## Useful Operating Notes

- The chip must be in sleep before changing `PRESCALE`
- After writing `PRESCALE`, clear `SLEEP` to restart the oscillator
- `RESTART` may be set after wake-up and should not automatically be treated as an error when verifying `MODE1`
- Multi-byte LED register reads and writes should use `AI` auto-increment in `MODE1`
- A `50 Hz` servo-style output corresponds to a `20 ms` frame
- With the nominal `25 MHz` oscillator, a prescale of `121` (`0x79`) is the normal value for about `50 Hz`

## Bench Findings Worth Reusing

- Basic register communication can succeed even when PWM output bring-up is still incomplete
- `MODE2` and `PRESCALE` readback are a useful first indicator that writes are sticking
- LED block readback can look wrong if multi-byte access is used without `AI`
- A readback such as `MODE1=0xB1` can still be acceptable if the expected operating bits match and `RESTART` is the only extra bit

## Known Quirks

- The internal oscillator and prescale sequencing are easy to get wrong during first bring-up
- Verifying `MODE1` too strictly can create false failures if `RESTART` is set
- Output enable and downstream power quality are separate concerns from register access; valid I2C communication does not guarantee that a connected servo or load will behave well

## Open Questions

- How close the actual oscillator is to the nominal `25 MHz` value on this board population
- Whether long-term use on this hardware needs additional attention to output enable handling, decoupling, or supply isolation for servo loads

## Verified vs Inferred

Verified from datasheet and bench behavior:

- register identities for `MODE1`, `MODE2`, `PRESCALE`, and LED channel blocks
- sleep-before-prescale sequencing
- need for `AI` during multi-byte LED register transactions
- `121` as the practical prescale value for about `50 Hz`
- `RESTART` can appear in `MODE1` during normal operation after wake-up

Inference or board-level follow-up:

- any remaining instability under servo load should be treated as a power, grounding, enable, or board-integration issue rather than a basic PCA9685 register-access issue unless new evidence shows otherwise
